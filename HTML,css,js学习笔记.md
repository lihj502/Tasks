首先，什么是网站呢？

网站就可以理解为在遵守因特网的规则下，使用HTML等制作的网页集合。

网页最通俗的理解就是网站的一“页”，我通过对HTML的学习才能够理解，一个网页就是一个HTML格式（大多以.html结尾）的文件，而它的识别则需要通过编译器“浏览器“来实现（就是不知道这么说是不是正确）。

## HTML学习——正文

### 对HTML的初印象

HTML又叫做”超文本标记语言“，它可以通过编写代码来对网页进行描述。

> 千万不要看到语言就想到的是编程语言，HTML是一种标记语言[^1]
>
> [^1]:标记语言包含的是一套标记标签，例如网页上的文字显示需要文字标签，网页上的图片显示需要图片标签，还有音频标签等等。

因为HTML可以加入图片、声音、动画等内容，它的内容超越了文本的限制 ，所以HTML被叫做”超文本语言“。

它被称为“超文本语言”的另一个原因是它可以从一个文件中跳转到另一个文件中，与网络上世界各地的文件相连接，而这一特点又被称为“超级链接文本”，所以同样简称为“超文本”。

### HTML之标签

#### 标签总述

标签给我的感觉可以说是html的核心所在，标签是由一对尖括号包裹的一小段具有特定含义可以被浏览器识别的单词所构成（例如<html>）

> 注意：虽然标签对于大小写<html> 和 <HTML>并不敏感，但是因为在万维网联盟的HML4的推荐标准中推荐是用小写，同时新版本的XHTML也被要求使用小写属性，因此我决定还是规规矩矩的使用小写输入。

标签分为两部分:，一部分是开始标签<html> ，另一部分是结束标签</html>, 处于两个标签之间的部分被称作“标签体”。

> 注意：虽然在练习中我发现即使忘记了加入结束标签，我的浏览器依然可以正常显示我的内容，但是好的习惯是很重要的，因为我在网上搜索这种情况的时候被警告，如果忘记使用结束标签可能会产生不可预料的结果或错误，因此还是要保持一个好的习惯（加上结束标签）才行。

13. 路径

   * 相对路径

     只需要通过特定的符号来表示目标图片与HTML文件的相对位置即可展示，共分为以下三种情况：

     1. 和HTML文件同一级路径——不需要任何特殊符号，直接粘贴文件名

     2. 在HTML文件下一级路径——需要`/`来表达

        > 首先粘贴和HTML文件在同一级的目标图片所在的文件夹名称，再使用`/`粘贴图片名称即可（如果图片还在更下一级，就继续重复最初的操作即可）

     3. 在HTML文件的上一级路径——需要`.../`来表示

        > 但是事实上我觉得这个应该不会常用，一般我都可能会把HTML文件放在根目录里面，应该不会出现出现上一级路径的情况

     > **总结**：总之这种表达方式比直接从网上复制地址要简洁美观，而且对图片的位置也很明显。

   * 绝对路径

     最通俗的解释是目标图片在本地电脑中的位置（此方法的局限性在于一旦将HTML文件发送给其他人，别人没有与你相同的目标图片所处位置的话将会无法显示图片）或者是完整的在网络上的地址（此方法的缺点是可能目标图片的网络地址很长，使得代码可读性减弱）

     > 如果手打网络地址的话一定要注意，网络地址的斜杠方向和本地文件的斜杠方向不同

14. 超链接标签（将一个页面链接到另一个页面）

   对于超链接语法中的每个属性的作用的记录：

   `<a href="链接目标的地址" target="链接地址的窗口的打开方式(又称为属性)">描述性的文字或者图像等都可以</a>`

   记忆的方法：对于各种缩写，尽量去思考查阅它的完全版单词，这样通过英语和HTML语法搭建起桥梁有助于对于各种不同的缩写的理解性记忆（例如，超链接中的a，就是anchor（锚，也就是链接）的缩写）

   对于targrt中有两种页面的打开方式可供选择：

   1. `-self`是在原页面直接打开超链接内容（同样也是不写target时的默认超链接打开方式）
   2. `-blank`是自动在一个新的标签页中打开超链接内容

   超链接标签的分类：

   1. 外部链接

      > 即链接的文件地址存在于互联网上

   2. 内部链接

      > 链接的文件和本HTML文件存在在一个目标文件夹中
      >
      > 因此链接的地址只需要输入将被链接的文件名称即可

   3. 空链接

      > 它的目的是在链接地址所在的网页尚未完善时，代替的假链接
      >
      > 在地址位置填入“#”即可

   4. 下载链接

      > 如果链接的地址是一个文件或者压缩包的话，就会自动下载这个文件

   当然，链接的地址可以是多样的，那么同样的，可设置超链接的对象也是多元化的，不仅仅是文字可以设置超链接，图像、表格、视频等都可以实现超链接。

    5. 锚点链接
    
       通俗理解就是在当前页面中可以跳转至指定的当前页面位置的一个链接。
    
       锚点链接的创建方式如下：
    
       * 找到将要跳转的目标地点，在这个地点的标题处添加一个id属性（即`<h3 id="xxx">``<h3>`）
    
         > ps:xxx是为这个id属性起的名字，可以随意起
    
       * 在超链接文本的地址处用“#xxx”来代替本应写的地址即可。

### HTML之块级元素

特点：在块级元素中，只有属于此块级元素的元素可以存在，其他的元素都无法和块级元素同行，可以看作一个盒子。

虽然可以通过css的修饰控制它的宽度和高度，但是即使它的宽度被限制在无法占满一行，依然没有元素可以和它在一行内共存。

类似`<p>``<h>`这样可以容纳文字类的块级元素无法在内部容纳<div>标签，即使强制将代码嵌套输入，在网页生成时，查看网页的源代码依然会发现<div>不在<p>标签中。

---

### HTML之行内元素

特点：行内元素又被称作内联元素或内嵌元素，和块级元素相比，它可以和其它元素和平共处于一行。

如果不使用`<br />`这种分行标签的话，即使在代码中分段，在网页中也会自动回到一行。

与块级元素相反，行内元素无法设置它的宽度和高度。

# CSS

## 对css的初印象

首先，css的中文全称是层叠式样式表，也被称为css样式表或者级联式样式表

它的主要目的是美化HTML的页面，是版面样式和布局更加美观。

总体的结构总的来说即为：将要被改样式的目标 { 将其改为什么样式 }

> 小tip：每一个属性的结尾都要加入分号

---

---

## css的引入方式

1. 内部样式表

   标签命名方式：`<style> 选择器{放置各种属性} </style>`

   > 将css的代码放在标签`<style></style>`中，然后将这个整体放入HTML代码的head中（但是其实可以将style标签放在HTML代码中的任意一个地方，但是为了方便阅读，因此统一将style放在head标签里面。）
   >
   > 控制范围是当前存在的HTML页面

2. 行内样式表

   标签命名方式：`<将要该改变的目标标签 style="放置css属性">`

   > 对于单个或者较少的段落需要css简单的修饰，则采用此种方式

3. 外部样式表（重点）

   适用于样式比较多的情况，将样式单独写入css文件中，之后将css文件引入到HTML页面中使用。

   1. 首先，先新建一个后缀名为.css的样式文件，将所有css代码都放入样式文件中。

   2. 在这个样式文件中无需写style标签，直接`选择器{放置各种属性}`

   3. 在head标签中使用link标签引入css

      > link标签中需要定义两个属性
      >
      > * rel，当前文件和被链接文件之间的关系，目前学到的只有指定为“stylesheet”，表示被链接的文档是一个样式表文件。

## css的浮动

相应的语法规范：`选择器 {float: 填写所需的属性值；}`

其属性值包含以下三个：

* none，元素不浮动
* left，元素向左浮动
* right，元素向右浮动



---

### 对浮动的理解

浮动属性主要用于创建浮动框，此时的浮动框将会做出相应的指示行动，直到行动到触及左边缘或右边缘或者另一个浮动框的边缘以后，浮动框将会结束相应的指令行动。



---

### css浮动的特性

1. 浮动的元素会脱离标准流的控制，移动到指定的位置
2. 浮动的元素移动后，原来的位置将不再存在，后面的标准流元素将自动填满浮动元素的位置。
3. 多个浮动元素会自动变为一行内进行显示，当屏幕界面变小无法容纳完整的一行盒子，即当前屏幕的宽度无法装下一整行盒子时，多出来的盒子会自动另起一行。



----

### css浮动技巧之搭配标准流父级

使用背景：由于在一些场景中，需要两个css的浮动框同时存在于整个HTML页面的顶部中央整齐排列，但是浮动只有左浮动或右浮动或者不动，因此无法达到所需的效果，所以出现了这种方法。

完成此效果的方法：

1. 制作一个空盒子div，并设置它与所需的浮动框相同的高度，宽度为两个浮动框的和
2. 通过css的修饰，将空盒子的位置设置为顶部居中
3. 接着在这个空盒子中添加两个向左或向右浮动的浮动框即可完成预期的效果

> 经过思考，我想到的此技巧的替代方法：
>
> 1. 先量出界面中所需的浮动盒子距离边框的白色宽度记为x
> 2. 创建一个白色的，高度与“将要处于整个HTML界面顶部中央的盒子”一样高向左浮动的盒子宽度为x
> 3. 再创建两个“将要处于整个HTML界面顶部中央整齐排列的盒子”即可同样完成所需的效果

### css浮动之清除浮动

此方法的应用背景：

在真正的网页制作中，对于一个盒子，如果为它设定了固定的高度和宽度属性，那么如果以这个盒子为模板，则在未来的网页制作中，一旦出现了内容超出盒子范围的情况，那么，内容的显示将出现问题。

此时就需要一个可以随着内容的改变而自由的更改高度盒子。

第一个想法是不为盒子设定高度，通过内部的内容增添自动撑开盒子。但是由于内部需要一个个小的浮动盒子为模板，在内部的盒子设置为浮动以后，原来的大盒子的高度就自动的变为了零。

方法：

1. 额外标签法

   > 通过在最后所有浮动框的最后面加入一个空的标签
   >
   > 可以使用过`<div style="clear:both"></div>`
   >
   > tip：要求新添加的标签必须是块级元素，不能是行内元素

2. 父级添加overflow属性

   > 语法格式一般为overflow：hidden或者auto或者scroll；

3. 父级添加after伪元素

4. 父级添加双伪元素

## js函数

js函数是由事件驱动或者当它被调用时执行的可重复使用的代码块，其中事件是指发生在 HTML 元素上的事情，一般来说当在 HTML 页面中使用 JavaScript 时， JavaScript 就可以触发这些事件。

js函数的语法就是一段包裹在花括号中的代码块，前面使用了关键词 function：

> 函数的调用：
>
> 在调用函数的同时可以向其传递参数值，而这些参数值可以在函数中使用。
>
> 参数一次发送的参数值数量任意，但要记得用逗号隔开
>
> 语法规范：myFunction(*argument1,argument2*)
>
> 如果向函数中输入函数值的话，一定要在声明函数的同时，将参数作为变量来声明。
>
> 语法规范：function myFunction(***var1***,***var2***){	代码	}
>
> tip：变量和参数必须以相同的顺序在调用的位置以及声明的位置出现。

和c语言相同，js的函数同样可以设定返回值，在return出现以后，函数会停止执行，并返回指定的值，而这时调用函数的位置将被函数的返回值取代。

>即使仅仅希望退出函数时 ，也可使用 return 语句，不过此时不需要添加额外的返回值。

在函数中的很多地方，js的语法和c语言都十分的相同，再比如，在 JavaScript 函数内部声明的变量（使用 var）就是局部变量，所以只能在函数内部访问它

>根据局部变量的特点我们完全可以在不同的函数中使用名称相同的局部变量，因为只要函数运行完毕，本地变量就会被删除。

那么说到局部变量，与之相对的一定是全局变量，在函数外声明的变量被称作全局变量，网页上的所有脚本和函数都能访问它，在页面关闭以后，变量将会自动删除。

> 非严格模式下即使是在函数中给未声明变量赋值依然可以创建全局变量，而该变量将被自动作为 window 的一个属性，可以用delet删除。

# html，css，js各自职能和关系

三者各自的职能如下表

| 标准 |                           说明                            |
| :--: | :-------------------------------------------------------: |
| 结构 |        结构用于对网页元素进行整理和分类——HTML负责         |
| 表现 | 表现用于设置网页元素的版式、颜色、大小等外形样式——css负责 |
| 行为 |        行为是指网页模型的定义及交互的编写——js负责         |

三者之间的关系我想到了一个比较有意思的比喻（来自从小学追火影的老火影迷脑洞）

可以将三者的关系比喻为火影中佐助的须佐能乎的三种形态：

* 第一形态——html

  “此时的半身须佐能乎只有简单的骨架以及肌肉，处于较为原始的状态

* 第二形态——css

  此时的半身须佐能乎出现了皮肤和盔甲，对第一形态做了外观上的修饰

* 第三形态——js

  此时的须佐能乎变为完全体，可以站立行动，完成了从静到动的转化

总结来说，只有在html，css和js三者的共同发挥作用下，才会出现我们现在看到的丰富绚丽的各式网站。
